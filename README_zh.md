# Desktop Hachimi - 智能桌宠伴侣 🐾

<div align="center">
    <img src="/ico/Desktop Hachimi ico.ico" width="150" height="150" />
</div>

Desktop Hachimi 是一个功能丰富的桌面宠物应用程序，为您的 Windows 桌面增添活力和趣味。这个可爱的数字伙伴会在您的桌面上自由活动，响应您的互动，陪您一起听音乐，为您的数字工作空间带来生气。

---

## 功能介绍

### 🐾 桌宠状态

桌宠会在四种状态之间自动切换，也会响应用户操作：

| 状态 | 说明 |
|------|------|
| **动感** | 活泼动画 —— 启动时、播放音乐时、鼠标靠近时触发 |
| **非移动（Idle）** | 静止动画 —— 桌宠停在原地时播放 |
| **移动（Move）** | 行走动画 —— 桌宠在屏幕上自由游走 |
| **拖拽（Drag）** | 拖拽动画 —— 用鼠标拖动桌宠时播放 |

状态每 5 秒根据可配置的权重（见 `weights.json`）随机切换一次。

### 🎵 音乐联动模式

内置音乐播放器与桌宠行为直接挂钩：

- 歌曲**开始播放**时，桌宠立即进入**动感**状态，并在整首歌播放期间保持该状态。
- 歌曲**停止或暂停**时，桌宠解除锁定，根据配置的权重随机切换到其他状态（非移动、移动或动感）。
- 音乐锁定期间，其他所有状态切换逻辑均不会打断动感状态，包括：5 秒自主切换定时器、鼠标跟随逻辑、移动触发等。

### 🎵 音乐播放器

右键菜单 → **音乐播放器**，或从系统托盘打开。

- 支持格式：MP3、WAV、OGG、FLAC、AAC、M4A、WMA
- 播放模式：循环全部 / 单曲循环 / 顺序播放（不循环）
- 控制按钮：上一首、播放/暂停、下一首
- 双击曲目可直接跳转播放
- 从电脑任意位置添加歌曲（自动复制到 `Music/` 文件夹）
- 删除曲目（移至回收站）

### 🖱️ 鼠标跟随

开启后，桌宠会追随光标在屏幕上移动。光标靠近时切换为动感状态；光标远离后，桌宠恢复移动状态继续游走。

### 🎨 桌宠自定义

- **切换桌宠** —— 从 `Pets/` 文件夹下的所有桌宠中选择
- **创建桌宠** —— 内置向导，用自己的 GIF 和图标组装新桌宠
- **删除桌宠** —— 从菜单将桌宠文件夹移至回收站
- **桌宠大小** —— x0.1 ~ x2.0，步进 0.1
- **透明度** —— 10% ~ 100%，步进 10%
- **速度** —— 10 档可调
- **状态权重** —— 通过权重编辑器调整各状态的出现概率
- **运动方向反转** —— 为每个移动变体单独配置向非默认方向运动时是否水平翻转精灵图

### 🖥️ 显示

- **最上层显示** —— 桌宠始终显示在所有窗口之上
- **多显示器支持** —— 桌宠会被限制在当前所在屏幕的范围内
- **位置记忆** —— 窗口位置在两次运行之间自动保存与恢复

### ⚙️ 系统

- **开机自启动** —— 随 Windows 启动
- **系统托盘图标** —— 无需打开任何窗口即可访问完整菜单
- **检查更新** —— 内置于"关于"对话框

## 目录结构

```
Desktop-Hachimi/
├── main.py
├── requirements.txt
├── config.json              <- 自动生成，保存用户设置
├── core/                    <- 核心逻辑（配置、GIF 加载、桌宠数据）
├── ui/                      <- UI 组件（桌宠窗口、音乐播放器）
├── compat/                  <- 平台适配（自启动、DPI、回收站）
├── ico/                     <- UI 及窗口图标
├── Music/                   <- 音乐文件夹（在此添加曲目）
└── Pets/
    └── Ameath/              <- 默认桌宠文件夹
        ├── Ameath.ico       <- 桌宠图标
        ├── Ameath.gif       <- 动感状态动画
        ├── drag.gif         <- 拖拽状态动画
        ├── idle.gif         <- 非移动状态（单图）
        │   或 idle1.gif, idle2.gif ...
        ├── move.gif         <- 移动状态（单图）
        │   或 move1.gif, move2.gif ...
        ├── weights.json     <- 状态权重
        └── flip.json        <- 运动方向反转配置（可选）
```

---

## weights.json 格式示例

```json
{
  "dynamic_weight": 3,
  "idle_weight": [2],
  "move_weight": [1]
}
```

若有多个 idle/move GIF，weight 数组长度须与文件数量一致：

```json
{
  "dynamic_weight": 3,
  "idle_weight": [2, 3],
  "move_weight": [1, 2, 1]
}
```

---

## flip.json 格式示例

```json
{
  "move": {
    "enabled": true,
    "default_dir": "left"
  },
  "move2": {
    "enabled": true,
    "default_dir": "right"
  }
}
```

`default_dir` 为 `"left"` 时：向左运动不翻转，向右运动翻转图片。

## 右键菜单说明

右键点击桌宠或系统托盘图标均可打开菜单：

| 菜单项 | 功能 |
|--------|------|
| 切换桌宠 | 从 `Pets/` 下的所有桌宠中选择 |
| 删除桌宠 | 将桌宠文件夹移至回收站 |
| 桌宠大小 | x0.1 ~ x2.0，步进 0.1 |
| 透明度 | 10% ~ 100%，步进 10% |
| 速度 | 1 ~ 10 档 |
| 鼠标跟随 | 开启后桌宠跟随鼠标移动 |
| 最上层显示 | 桌宠显示在所有窗口最前 |
| 开机自启动 | 注册或移除 Windows 自启动项 |
| 音乐播放器 | 打开音乐播放器窗口 |
| 调整状态权重 | 修改各状态的出现概率权重 |
| 调整运动方向反转 | 为每个移动变体配置翻转行为 |
| 创建桌宠 | 打开桌宠创建向导 |
| 关于 | 软件信息与更新检查 |
| 退出 | 关闭程序 |
